## Presentation Tips

### Timing
| Section | Time |
|---------|------|
| Problem & Motivation | 2 min |
| Solution Overview | 3 min |
| Technical Deep-Dive | 5 min |
| Demo | 3 min |
| Results | 3 min |
| Lessons & Future | 2 min |
| Q&A | 5 min |
| **Total** | **~23 min** |

### Do's
- ✅ Practice your demo multiple times
- ✅ Have backup slides for anticipated questions
- ✅ Use large fonts (24pt minimum)
- ✅ Include page numbers
- ✅ One main point per slide
- ✅ Use visuals over text

### Don'ts
- ❌ Read from slides
- ❌ Include code that's too small to read
- ❌ Have more than 6 bullets per slide
- ❌ Rush through the demo
- ❌ Skip the problem motivation

### Demo Backup Plan
1. Have video recording as backup
2. Have screenshots ready
3. Test demo 30 minutes before presentation
4. Have fallback if network is slow

## Slide Design Guidelines

### Colors
- Use consistent color scheme
- High contrast for readability
- Limit to 3-4 colors

### Fonts
- Sans-serif for readability (Arial, Helvetica)
- Title: 36-44pt
- Body: 24-28pt
- Min readable: 18pt

### Images
- High resolution
- Consistent style
- Meaningful (not decoration)

### Code Snippets
- Syntax highlighting
- Max 10-15 lines
- Focus on key parts
